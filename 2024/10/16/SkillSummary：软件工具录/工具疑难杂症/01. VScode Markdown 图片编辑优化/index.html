《工具疑难杂症》VScode Markdown 图片编辑优化

发表于2024-10-16更新于2024-10-16

虽然 VScode Markdown 编辑时自带图片保存功能，但无法自定义图片保存名。所以研究了一下如何优化图片保存。

1：相关环境

项目 版本
操作系统 Win 10
Visual Studio Code 1.83.1 (system setup)
Paste Image 插件 v1.0.4

2：具体操作

要根据当前时间格式化命名图片，VScode 默认不支持此功能。所以需要借助 Paste Image 插件。

安装 Paste Image 插件。

打开插件扩展设置。

2.1：时间格式化命名图片

配置项 默认值
Default Name Y-MM-DD-HH-mm-ss

2.2：图片文件保存位置

配置项 默认值
Path ${currentFileDir}

根据需要，将图片保存到当前 md 文件所在目录下的 /assets/ 路径中。
修改配置值为 ${currentFileDir}/assets/。

2.3：插入图片默认描述信息

配置项 默认值
Insert Pattern ${imageSyntaxPrefix}${imageFilePath}${imageSyntaxSuffix}

以图片名作为默认描述：![${imageFileNameWithoutExt}](${imageFilePath})

将一段文字作为默认描述：![Alt text](${imageFilePath})
